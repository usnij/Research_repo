## 1. 배경 및 목적

LZT(LinearZThickness) 등 3DGUT 렌더링 개선 기법의 효과를 공정하게 평가하기 위해, 표준 벤치마크 데이터셋 전체에 대한 일괄 학습·평가 파이프라인이 필요하다. bonsai 단일 씬 실험만으로는 일반화 가능성을 판단하기 어렵기 때문이다.

### 데이터셋 선정 근거

논문(3DGS, 3DGUT)에서 공통으로 사용하는 표준 벤치마크 3종 13개 씬을 채택했다.

| 데이터셋 | 씬 수 | 씬 목록 |
|---------|------|--------|
| **MipNeRF-360** | 9 | bicycle, flowers, garden, stump, treehill (outdoor) / bonsai, counter, kitchen, room (indoor) |
| **Tanks & Temples (T&T)** | 2 | train, truck |
| **Deep Blending (DB)** | 2 | drjohnson, playroom |
| **합계** | **13** | |

## 3. Downsample Factor 표준

### 논문 기준

**3DGS 논문** (arXiv:2308.04079) 및 **3DGUT 논문** (arXiv:2412.12507) 공통 기준:

| 데이터셋 | 씬 | downsample factor |
|---------|---|------------------|
| MipNeRF-360 Outdoor | bicycle, flowers, garden, stump, treehill | **4** |
| MipNeRF-360 Indoor | bonsai, counter, kitchen, room | **2** |
| Tanks & Temples | train, truck | **1** (native resolution) |
| Deep Blending | drjohnson, playroom | **1** (native resolution) |

> T&T와 DB는 원본 해상도 사용. 1600px 초과 시 자동 축소.
> 3DGUT 논문은 DB를 평가하지 않았으나, 3DGS 논문 기준(factor=1)을 따름.

## 4. Baseline K 설정

### K=0 선택 근거

`configs/render/3dgut.yaml`의 기본값은 `k_buffer_size: 0 # 0 means unsorted`이다.

bonsai 씬 실험 결과:

| K | PSNR | 비고 |
|---|------|------|
| K=0 | 32.352 dB | **config 기본값, 3DGUT 논문 보고치** |
| K=4 | 32.249 dB | k-buffer 활성화 (-0.103 dB) |
| K=16 LZT | 31.989 dB | 최고 LZT 결과 |

K=4 baseline은 k-buffer 자체의 영향을 분리하기 위해 중간 실험에서만 사용했다. 논문 비교용 baseline은 **K=0** (vanilla 3DGUT, 기본 설정).

## 5. 벤치마크 스크립트 구성

### 파일 위치

```
benchmark/
  all13_train.sh     ← 13개 씬 일괄 학습
  all13_render.sh    ← 학습 완료 후 일괄 평가
  all13_stats.py     ← PSNR / SSIM / LPIPS / ms 출력
```

### 디렉토리 구조

```
results/
  baseline/            ← vanilla 3DGUT (K=0)
    .cache/            ← JIT 캐시 (씬 간 공유)
    train_bicycle.log
    train_bonsai.log
    ...
    render_bicycle.log
    ...
    bicycle/
      ckpt_last.pt
      eval/
        renders/
        gt/
    bonsai/
      ...

  lzt_k16/             ← LZT K=16
    ...

  lzt_k24/             ← LZT K=24
    ...
```

### 사용법

```bash
# 1. 학습 (baseline)
bash benchmark/all13_train.sh apps/colmap_3dgut

# 2. 학습 (변형 실험, 예: LZT K=16)
RESULT_DIR=results/lzt_k16 bash benchmark/all13_train.sh apps/colmap_3dgut \
  render.splat.linear_zthickness=true render.splat.k_buffer_size=16

# 3. 평가
bash benchmark/all13_render.sh results/baseline

# 4. 결과 출력 (PSNR / SSIM / LPIPS / ms/frame)
python benchmark/all13_stats.py results/baseline
```

### 주요 설계 결정 사항

| 항목 | 결정 | 이유 |
|------|------|------|
| JIT 캐시 위치 | `$RESULT_DIR/.cache` | 실험별 분리, 충돌 방지 |
| `--compute-extra-metrics` | **사용 안 함** | `action="store_false"` → 붙이면 SSIM/LPIPS 꺼짐 |
| 결과 파싱 | Python (`all13_stats.py`) | awk보다 안정적, LPIPS까지 파싱 |
| 기존 `print_stats.sh` | 사용 안 함 | LPIPS 미지원, awk 파싱 불안정 |

## 6. `--compute-extra-metrics` 플래그 주의사항

`render.py`의 해당 인수 정의:

```python
parser.add_argument(
    "--compute-extra-metrics",
    action="store_false",   # 플래그를 붙이면 False로 설정됨!
    help="If set, extra image metrics will not be computed [True by default]",
)
```

- **플래그 없음** (기본값): `compute_extra_metrics=True` → PSNR + SSIM + LPIPS 모두 계산
- **`--compute-extra-metrics` 붙임**: `compute_extra_metrics=False` → PSNR만 계산

→ 벤치마크 render 스크립트에서는 이 플래그를 **붙이지 않는다**.
