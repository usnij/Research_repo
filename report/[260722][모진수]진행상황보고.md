# [260722][모진수] 오늘 수행한 것

## 1. 오늘의 연구 진행 상황 및 수행한 것

오늘 진행한 것은 크게 다음 두 가지이다.

1. EVER overlap compositing을 근사하는 구조의 구현, 학습, 분석 결과 정리
2. ray tracing 기반 EVER 구조에서 고해상도 학습 비용을 줄이기 위한 근본 방향 검토

1번의 내용을 더 자세히 말하면, 현재 연구는 단순히 이미 학습된 EVER scene에 근사 inference만 적용하는 것이 아니라, **scene 자체가 근사 구조에 잘 맞는 Gaussian 분포로 수렴하도록 학습시키는 것**을 목표로 한다.

EVER는 overlap을 event 단위로 정확하게 처리하지만, 현재 근사 구조는 여러 Gaussian interval을 하나의 cluster로 묶어 처리한다. 따라서 cluster가 되어도 품질이 크게 떨어지지 않는 Gaussian 분포가 필요하다고 판단했고 이를 통해 근사구조를 통한 품질을 최대한 보존하며 근사구조 특유의 속도를 보장할 수 있을것이라고 추정했다. 


처음에는 단순히 씬에서 근사했을 때 품질이 떨어지는 overlap형태를 줄이는 것을 목표로 했다. 여기서 품질이 떨어는 overlap형태를 bad-overlap이라고 부르겠다.

그래서 학습된 모델에 대해 regularization loss를 부여해 bad-overlap의 분포가 줄어드는 것을 목표로 loss설계를 시도했으나 잘 되지 않아 조금 더 구체적으로 설계해야함을 판단했다.

근본적으로 가우시안을 통한 cluster의 분포 자체가 길이/밀도/색이 너무 다른 구조가 안 되게 loss 설계를 더욱 정교하게 해야 한다고 판단된다.

그래서 cluster 내부 길이/밀도/색 불균질성을 줄이려는 loss를 설계 중에 있다. 이를 통해 학습된 모델의 가우시안 분포가 근사친화적이게 되는 것을 목표로 한다. 

## 2. 이전까지 수행된 내용

이전까지 확인한 핵심 내용은 다음과 같다.

- payload에 한 번에 담을 수 있는 interval 수를 늘리면서 근사 구조의 품질이 크게 개선되었다. 원본 ever는 entry, exit정보가 모두 payload에 담겨야 하지만 근사구조는 entry정보만 있으면 됨
- 기존에는 근사 forward를 학습된 모델에 대해 inference-only로 적용했지만, 현재는 forward/backward를 맞추고 finite-difference 검증까지 거쳐 학습 가능한 상태가 되었다.
- 근사 렌더러로 full training은 가능하지만, exact EVER 수준의 품질에는 아직 도달하지 못했다.
- 큰 오차는 단순히 Gaussian 수가 부족해서가 아니라, 하나의 cluster 안에 길이/밀도/색이 다른 Gaussian이 섞이는 구조에서 강하게 발생한다.

기준이 되는 모델과 수치는 다음과 같다.

| 항목 | 내용 |
| --- | --- |
| Exact 기준 모델 | `output/ficus_no_mask` |
| Exact 기준 품질 | PSNR 34.733, SSIM 0.9849, L1 0.004782 |
| 현재 주요 근사 학습 모델 | 근사 학습 기본안 |
| 현재 근사 학습 품질 | PSNR 32.233, train PSNR 33.691 |
| 현재 문제 | exact 대비 약 2.5dB 낮고, heterogeneous cluster에서 오차가 집중됨 |

보고서에서 사용하는 이름은 내부 실험명이 아니라 아래처럼 직관적인 명칭으로 통일한다.

| 보고용 이름 | 의미 |
| --- | --- |
| 기본 근사구조 | overlap interval을 cluster 처리한 구조 |
| 근사 친화성 loss | cluster 내부 길이/밀도/색 불균질성을 줄이려는 loss |

## 3. 근사 친화성 loss 설계 방향

현재 근사 구조의 핵심 문제는 overlap 자체가 아니라, 하나의 cluster로 묶였을 때 서로 다른 성질의 Gaussian들이 하나의 uniform medium으로 평균화된다는 점이다. 즉, cluster 내부 Gaussian들의 길이, 밀도, 색이 크게 다르면 근사 렌더러는 그 내부 순서와 세부 변화를 표현하지 못하고 품질이 떨어진다.

처음에는 품질이 떨어지는 overlap pair를 줄이는 방향으로 접근했다. 하지만 단순 pair-wise regularization은 충분하지 않았다. exact renderer 위에서 overlap을 줄이는 loss를 주면, exact 기준에서는 문제가 아닌 구조를 억지로 바꾸게 되어 photometric loss와 충돌할 수 있다. 따라서 현재는 pair를 직접 없애기보다, **근사 렌더러가 하나의 cluster로 처리해도 무리가 없는 Gaussian 분포**를 만드는 쪽으로 방향을 바꾸었다.

근사 친화성 loss의 목표는 다음과 같다.

| 제어 대상 | 의미 | 기대 효과 |
| --- | --- | --- |
| cluster 내부 길이 불균질성 | 같은 cluster 안 interval 길이가 지나치게 다른 정도 | 하나의 uniform medium으로 근사할 때 생기는 길이 왜곡 감소 |
| cluster 내부 밀도 불균질성 | 같은 cluster 안 Gaussian density가 들쭉날쭉한 정도 | optical thickness가 특정 Gaussian에 과도하게 쏠리는 현상 완화 |
| cluster 내부 색 불균질성 | 같은 cluster 안 Gaussian 색이 크게 다른 정도 | cluster color 평균화로 생기는 색 번짐 감소 |

현재 설계의 중요한 점은 색 자체를 직접 평균화하지 않는 것이다. 색을 직접 평균화하면 exact 품질은 유지되어도 근사 품질이 개선되지 않거나, 반대로 표현력이 줄어들 수 있다. 따라서 색 차이는 “이 cluster가 위험한가”를 판단하는 gate로 사용하고, 실제 gradient는 길이와 밀도 구조를 조정하는 쪽으로 흐르게 하는 방향을 사용했다.

다만 이 설계는 아직 완성된 형태가 아니다. 실제 300iter 적용 실험에서는 길이/밀도/색 불균질성은 줄었지만, 전체 optical thickness가 증가하고 특정 Gaussian에 mass가 쏠리면서 exact 품질이 크게 하락했다. 따라서 다음 버전에서는 cluster를 균질하게 만드는 항뿐 아니라, optical mass 보존 항을 반드시 함께 넣어야 한다.

정리하면 현재 근사 친화성 loss의 방향은 다음과 같다.

```text
근사 친화성 loss v1:
  cluster 내부 길이/밀도/색 불균질성 감소

문제:
  전체 optical thickness 증가
  특정 Gaussian으로 mass 쏠림
  exact 품질 훼손

다음 방향:
  불균질성 감소 + optical mass 보존 + 지배 Gaussian 억제
```

## 5. 해석

이번 결과에서 확인된 점:

1. payload 용량 개선은 근사 품질에 결정적인 영향을 주었다.
2. 이월 기반 cluster forward/backward는 구현상 학습 가능한 수준으로 정리되었다.
3. 근사 full training은 가능하며, 근사 학습 기본안은 12.77분에 32.23dB를 달성했다.
4. exact EVER 34.73dB와의 차이는 단순 primitive 수 부족만으로 설명하기 어렵다.
5. 오차가 큰 ray는 large active set, large cluster, color/density/length 불균질성이 강하다.
6. color-only rewrite나 opacity redistribution은 근사 오차를 줄이는 핵심 레버가 아니었다.
7. 근사 친화성 loss는 구조 지표를 움직일 수 있지만, tau_total과 dominant_tau_ratio를 증가시켜 exact 품질을 크게 망가뜨렸다.

실패하거나 예상과 달랐던 점:

1. exact forward 위에 overlap 제어 regularizer를 얹는 방식은 no-reg 대조군 대비 PSNR 개선이 없었다.
2. geometry loss는 position을 직접 움직이지 못하고 ray 방향 scale/rotation에만 주로 작용했다.
3. cluster를 균질하게 만드는 loss는 작동했지만, optical mass 보존 없이 적용하면 scene 자체가 훼손되었다.
4. 근사 친화성 loss를 처음부터 강하게 켜면 primitive 폭주나 품질 하락이 나타났다.

원인 추정:

- 근사 오차는 단순히 cluster 크기 하나로 설명되지 않는다.
- uniform medium 근사는 cluster 내부의 length, density, color, tau 분포가 같이 안정적일 때만 잘 작동한다.
- 현재 loss는 length/density/color 불균질성을 줄이는 데는 성공했지만, 그 과정에서 전체 optical thickness와 dominant Gaussian 비율을 제어하지 못했다.
- 따라서 다음 loss는 cluster heterogeneity 감소와 optical mass 보존을 동시에 만족해야 한다.

## 6. 다음 계획

다음으로 진행할 작업:

1. 근사 친화성 loss v2 설계
2. tau 보존 항과 dominant tau ratio 억제 항 추가
3. 근사 학습 기본안의 densification regime 위에서 약한 late-stage regularization 실험
4. top-error cluster 구조를 더 세분화하여 어떤 지표를 줄이면 근사 PSNR이 실제로 오르는지 확인
5. 근사 친화적 densification/pruning 전략 설계

우선순위:

| 우선순위 | 작업 | 이유 |
| ---: | --- | --- |
| 1 | `L_tau_preserve` 설계 | 현재 근사 친화성 loss는 전체 optical thickness를 증가시켜 품질을 훼손함 |
| 2 | `L_dominant_tau_ratio` 설계 | cluster mass가 한 Gaussian에 쏠리는 현상을 억제해야 함 |
| 3 | 약한 late-stage 근사 친화성 loss schedule | 처음부터 강하게 걸면 학습과 densification이 불안정해짐 |
| 4 | 근사 학습 기본안 기준 재실험 | 현재 가장 안정적인 approximate-training baseline |
| 5 | cluster-aware densification 검토 | 최종적으로 Gaussian 생성 단계에서 근사 친화성을 유도해야 함 |

다음 loss 설계 초안:

```text
L_total =
  L_render
+ lambda_cluster * L_cluster_length_density_color
+ lambda_tau * L_tau_preserve
+ lambda_dom * L_dominant_tau_balance
```

이때 핵심은 cluster를 무조건 작게 만들거나 overlap을 무조건 제거하는 것이 아니다. 하나의 cluster로 묶여도 uniform medium 근사가 성립하도록, cluster 내부의 길이/밀도/색/tau 구조를 함께 안정화하는 것이다.
